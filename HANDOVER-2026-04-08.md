## Known Bugs (Unfixed)

### 1. BYPASS_AUTH Task Spam — PARTIALLY FIXED
**What happened overnight:** Daemon ran old code (started before fixes). Progress monitor created BYPASS_AUTH task every 5 min → failed with "Credit balance too low" → didn't count as completed → recreated. 100+ spam tasks.
**Fixed:** Progress monitor now counts `failed` tasks. `oneShot: true` flag set. BUT: the root cause is also that `claude -p` hits credit limits at night. Need rate-limit awareness in the scheduler.
**Remaining work:** Scheduler should check `isRateLimited()` before creating progress-monitor tasks.

### 2. StixDB Not Starting
**What:** Python dependency issue, never investigated.
**Impact:** Cross-agent memory (StixDB) not available. Agents can't share findings via graph DB.

### 3. OpenAI Key Loading
**What:** Health check says "OPENAI_API_KEY: Missing" but `.secrets.json` has the key. The health check runs before `cli.ts` loads `.secrets.json` into `process.env`. The actual codex-review agent uses `getSecretOrNull()` which reads `.secrets.json`, so it works at runtime.
**Fix needed:** Load `.secrets.json` in daemon startup, not just CLI entry point.

## Key Architecture Decisions

- **Engineering uses `claude -p`** (CLI subscription, not API credits) for code execution
- **No tool restrictions** on CLI — engineering needs full bash for build/test/deploy
- **Feature branches only** — all Organism code changes on `organism/<id>/<slug>` branches
- **Safety at orchestrator gate** — blocked patterns prevent task creation, not just execution
- **Cascade is unidirectional** — security → engineering only, no reverse cascade
- **Progress monitor objectives are one-shot or 7-day recurring** — not 3-day

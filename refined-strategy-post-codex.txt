3. REVIEW PIPELINE — RISK-BASED LANES
=======================================

Instead of 7 serial gates for everything, a risk classifier (cheap Haiku call) sorts
each task into one of 3 lanes:

LOW RISK (50% of tasks)
  Internal docs, data updates, routine content
  -> Quality Agent (1 pass) -> Auto-ship

MEDIUM RISK (35% of tasks)
  Code changes, marketing content, feature additions
  -> Grill-Me -> Quality Agent -> Codex Review -> Auto-ship

HIGH RISK (15% of tasks)
  Deploys, spending, legal/security, customer-facing, pricing, auth changes
  -> Grill-Me -> Quality Agent -> Copyright -> Legal -> Security
  -> Quality Guardian (full 6-phase audit) -> Codex Review
  -> G4 Board Gate (Rafael approves)

Key: Most tasks DON'T go through the full pipeline. Only ~15% get the full treatment.

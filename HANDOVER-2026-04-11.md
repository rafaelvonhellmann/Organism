# Organism Autonomy Hardening Handover

Date: 2026-04-11
Branch: `codex/review-organism-autonomy`

## What shipped

- Structured runtime failure mapping now covers:
  - `rate_limit`
  - `overload`
  - `timeout`
  - `missing_secret`
  - `auth_failure`
  - `tool_failure`
  - `policy_block`
  - `transport_error`
- Orphaned `running` sessions and `in_progress` tasks are recovered on daemon startup.
- Recovery now writes a checkpoint artifact and reschedules retryable work instead of abandoning it.
- Orchestrator submission failures now update structured runtime state instead of falling back to a free-text hard fail path.
- Sensitive actions are now blocked by project policy and autonomy mode, not hardcoded forever in the orchestrator.
- Goal dedupe now supports source-specific seeds, and duplicate active goal tasks are blocked.
- Organism task routing now respects the project roster even for the `organism` project itself, so non-core specialists do not get selected just because their keywords match.
- Palate source injection now uses the same project-capability filter as routing, and its cache now lives under the canonical external state directory.
- Git watcher now:
  - stores state in the canonical external state directory
  - establishes a baseline on first sight instead of generating startup spam
  - routes signals as `git_watcher` instead of generic monitor noise
- Runtime UI now shows:
  - current model backend
  - current code executor
  - web-search availability
  - paused/retrying workload
  - rate-limit state
- Autonomy health now reports when active core agents do not yet have 10 shadow runs recorded.
